1.  部署tokenModules模块：
    执行命令：npx hardhat ignition deploy ignition/modules/tokenModules.ts --network sepolia
    命令说明：部署以下合约WLTCMock,USDCMock, StableToken, MultiLeverageToken四个合约
2.  部署coreModules模块： 
    执行命令：npx hardhat ignition deploy ignition/modules/coreModules.ts --network sepolia
    命令说明：部署以下合约InterestManager, LTCPirceOracle, CustodianFixed, LinearDecrease, AuctionManager, LiquidationManager
3.  铸币USDC,WLTC合约： 
    执行命令：npx tsx scripts/1_mint_USDCWLTCtokens_allusers_viem.ts 
    命令说明：测试环境，部署者给开发组四个人铸币usdc(12000万枚) wltc(100万枚)
             仅有部署者可以铸币wltc,usdc
4.  铸币S token & L token:
    执行命令：npx tsx scripts/2_mint_SLtokens_viem.ts 
    命令说明：部署者给自己账户铸币，三种杠杆(CONSERVATIVE,MODERATE,AGGRESSIVE)分别铸币24.5万枚wltc, 24.5万枚wltc, 1万枚wltc；铸币价格P0=120;
             当前持仓：wltc(50万枚)，usdc(12000万枚),stable(9746666枚)，leverage(id5=2612万，id8=2352万，id2=60万)
             所有用户均可调用CustodianFixed.mint使用自己账户的wltc给自己铸币，mint前需要approve给custodianFixed相应数量的wltc
5.  在uniswap上创建wltc-usdc的池子：
    池子参数：V3 position; token0:wltc,token1:usdc(顺序由uniswap决定),fee:0.3%, full range;
    流动性参数：注入wltc(30万枚)和usdc(3600万枚),wltc初始价格为P0=120； 账户剩余20万wltc, 8400万usdc
    部署后地址：
        POOL:   0xCa250B562Beb3Be4fC75e79826390F9f14c622d0(每次重新部署会变化)
        Router: 0x3A9D48AB9751398BbFa63ad67599Bb04e4BdF98b(不变)
        Quoter: 0xEd1f6473345F45b75F8179591dd5bA1888cf2FB3(不变)
6.  部署ammModules模块：
    执行命令：npx hardhat ignition deploy ignition/modules/ammModules.ts --network sepolia
    命令说明：部署以下合约AMMLiquidaity, AMMSwap
7.  合约验证：
    执行命令：npx tsx scripts/3_verify_allContracts_viem.ts
8.  合约交互：
    检验交易：
        执行命令：npx tsx scripts/4_interact_amm_viem.ts
        命令说明：
            若流动性不足，则添加流动性, 注入流动性：500万usdc,500万stable; 需要向AMMLiquidity中approve相应数量的usdc和stable
            若Oracle价格无效，使用更新为uniswap中当前wltc的价格;
            Stable token<--> USDC token   (基于AMM      )
            Leverage token<--> USDC token (基于AMM + DEX)
    检验清算：
        执行命令：npx tsx scripts/5_auction_viem.ts
        命令说明：
            给CustodianFixed转入10万个stable, 用于奖励
